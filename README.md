# LSTM Uncertainty Study — ICU Mortality Prediction

Empirical study on the impact of missing values on uncertainty-aware LSTM models for clinical time series, using the **PhysioNet/Computing in Cardiology Challenge 2012** dataset.

This repository accompanies the bachelor's final project report:
> *The Impact of Missing Values on Uncertainty-Aware LSTM Models for Clinical Time Series: An Empirical Study on ICU Mortality Prediction*
> — Bruno Anderson Magalhães Rocha, Universidade do Porto (FCUP/FEUP), 2026.

## 1 — Download the dataset

The raw data must be obtained from PhysioNet (free registration required):

**[https://physionet.org/content/challenge-2012/1.0.0/](https://physionet.org/content/challenge-2012/1.0.0/)**

Download and extract the archive. Then copy the following folders into the `ds/` directory of this repository, so the final layout matches the tree above:

| PhysioNet folder | Destination in repo |
|---|---|
| `set-a/` | `ds/set-a/` |
| `set-b/` | `ds/set-b/` |
| `set-c/` | `ds/set-c/` |
| `outcomes/` | `ds/outcomes/` |

Each `set-*` folder contains one `.txt` file per patient (≈ 4,000 patients per set).
The `outcomes/` folder contains `Outcomes-a.txt`, `Outcomes-b.txt`, and `Outcomes-c.txt`.

## 3 — Running the notebooks

Run the three notebooks **in order**, once each:

### `dsPrep.ipynb` — Data preparation
Consolidates the raw PhysioNet patient files across sets A, B, and C; performs feature selection (7 clinical parameters over the first 12 hours of ICU stay); applies missingness-based cohort filtering; and produces the stratified 80/20 train/test split.

**Output:** `ds_lstm/trainLSTM.csv` and `ds_lstm/testLSTM.csv`

---

### `dtLstm.ipynb` — Model training ⚠️ *This step can take a significant amount of time*
Trains the three LSTM-based architectures — Standard LSTM, Deep Ensemble (5 members per fold), and Bayesian LSTM (variational inference) — under a 5×3 Repeated Stratified K-Fold cross-validation scheme (15 folds total, 75 networks for the ensemble alone). Includes loss-curve and ROC/PR-curve analysis across best, median, and worst folds.

**Output:** trained model checkpoints and metrics saved in `models2/`

> **Note:** Training time depends heavily on available hardware. On Apple Silicon (MPS) expect several hours for the full pipeline; on CPU it will be considerably longer. The standard and Bayesian models are the most time-consuming due to the variational inference overhead in the latter.

---

### `results.ipynb` — Evaluation and analysis
Loads the trained models and evaluates them on the held-out test set under nine controlled missingness scenarios (0% baseline + MCAR and Temporal-Block injection at 10%, 20%, 40%, and 60%). Computes discrimination metrics (AUC-PR, AUC-ROC), calibration metrics (Brier score, ECE), and 95% MC-Dropout credible-interval analysis. Generates all figures reported in the paper.

**Output:** `models_metrics.csv` (405 rows: 3 architectures × 9 scenarios × 15 folds) and all result plots.
